### 生成的创意

**[替代 #1]**: BMAD 学习浏览器
_概念_: 一个 Web 应用，将 `_bmad-output/` 目录下的所有产出物以结构化方式展示，团队成员可以浏览完整的 BMAD 工作流程和每个阶段的产物，替代直接看 Git 仓库里的 Markdown 文件。
_新颖性_: 不只是文档展示，而是让方法论本身可视化——团队成员能看到"为什么这样做"而不只是"做了什么"。

**[组合 #2]**: 交互式流程导航
_概念_: 工作流程图作为核心导航方式——点击"创建PRD"节点，右侧展开该阶段的产出文档，顶部显示使用的命令，底部显示演进时间线。
_新颖性_: 流程图、命令说明、产出物三者融为一体，学习者同时理解"做什么→怎么做→产出什么"。

**[组合 #3]**: 命令与产出映射
_概念_: 从 `bmad-help.csv` 提取每个工作流的命令、代理角色、说明，与实际产出文件自动关联。展示"📋 John(产品经理) 用 `bmad-bmm-create-prd` 产出了 → PRD文档"。
_新颖性_: 让方法论有了"人格"——每个代理角色的头像和职责都可见。

**[组合 #4]**: Git 演进时间线
_概念_: 通过 Git 历史追踪每份文档的版本变化，以时间线方式展示从初版到终版的演进过程，支持版本对比查看差异。
_新颖性_: 学习者能看到"一份好的 PRD 不是一次写成的"，理解迭代的价值。

**[适应 #5]**: Git 原生体验
_概念_: 借鉴 GitHub/GitLab 的交互模式，但专为 BMAD 文档优化。用 Markdown 渲染后的对比而非原始文本 diff。
_新颖性_: 团队零学习成本，用熟悉的 Git 心智模型理解 BMAD 的迭代过程。

**[修改 #6]**: 自适应双模式布局
_概念_: 全景模式→详情模式的自然切换。首页是 BMAD 完整流程图全景，每个阶段节点显示完成状态和摘要卡片。点击进入后切换为文档阅读视图，面包屑可随时返回。
_新颖性_: 既照顾"快速了解全局"的新手，又满足"深入某个阶段"的深度学习者。

**[消除 #7]**: 零数据库架构
_概念_: Go 后端直接读取项目目录结构，解析 Markdown 和 YAML/CSV 配置，内存缓存 + 文件监听热更新。无数据库、无登录、无编辑。
_新颖性_: 部署极简，`go run` 一行启动，团队成员零门槛访问。

**[反转 #8]**: 逆向追溯导航
_概念_: 默认展示当前最新进度，流程图上高亮"你在这里"，往回点击可追溯之前每个阶段的产出和决策。
_新颖性_: 符合实际工作心理——人们先关心"现在"，再追问"为什么"。

## 行动计划

**项目名称:** BMAD 学习浏览器 (BMAD Viewer)

**技术栈:**

3. 按 BMAD 流程完成：规划 → 架构 → 实施
